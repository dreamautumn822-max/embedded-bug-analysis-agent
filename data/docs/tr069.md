# TR-069 模块说明

## Inform 调用链

TR-069 inform 失败通常与 ACS 地址解析、DNS 缓存、认证参数、WAN 可达性有关。

完整链路包括读取 ACS 配置、DNS 解析、建立 TCP/TLS 连接、HTTP 认证、发送 Inform 和处理 ACS 响应。排查时应先确定失败发生在哪个阶段。

## DNS 缓存与 ACS 地址

DNS 配置更新后，TR-069 client 应清理 resolver cache，否则可能继续访问旧 ACS 地址。

如果日志中的解析结果与当前 ACS 域名记录不一致，或者重启 client 后恢复，应优先检查本地 resolver cache、DNS 更新通知和重连流程。

## 认证与证书

HTTP 401/403 通常需要检查 Connection Request 凭据、ACS 用户名密码和时间同步。TLS 握手失败需要检查 CA、证书有效期、SNI 和设备系统时间，不能简单归因于 WAN 不通。

## 现场排查与回归

现场至少采集 ACS URL、DNS 结果、连接错误、HTTP 状态码和 Inform 重试日志。回归应覆盖 DNS 切换、ACS 地址变更、认证失败恢复、证书更新和 WAN 重连。
