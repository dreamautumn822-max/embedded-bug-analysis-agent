# 固件升级与配置迁移说明

## 迁移原则

固件升级后出现网络异常时，需要检查配置迁移规则。旧版本配置 key 如果没有映射到新 schema，可能导致 DHCP 地址池、WAN 账号、Wi-Fi 参数丢失。

迁移过程应记录源版本、目标版本、执行规则、跳过字段和校验结果。对关键字段应提供默认值，但默认值不能静默覆盖用户已有配置。

## DHCP 地址池迁移

DHCP 地址池迁移需要确认 `lan.dhcp.pool_start` 和 `lan.dhcp.pool_end` 已从旧配置正确映射到新配置。

如果升级后每次启动都出现 `no free leases`，且 bridge 已经 ready，应优先检查地址池配置，而不是继续调整服务启动时序。

## WAN 与 Wi-Fi 配置迁移

WAN 迁移需要保留 PPPoE 用户名、密码、VLAN、MTU 和连接模式。Wi-Fi 迁移需要保留 SSID、加密方式、密码、信道和 Mesh 参数，并对废弃枚举值做显式转换。

## 校验、回滚与回归

迁移完成后应运行 Schema 校验和关键业务探测，并保存可审计的迁移报告。失败时应回滚配置或进入安全默认配置，避免设备处于半迁移状态。

回归测试至少覆盖跨两个历史版本升级、恢复出厂后升级、保留配置升级和升级失败回滚。
